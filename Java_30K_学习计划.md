# Java 30K 快速突击学习计划

## 📅 总体时间规划（3-6个月）

### 阶段一：基础巩固与框架精通（4-6周）
### 阶段二：分布式与微服务（4-6周）
### 阶段三：高并发与性能优化（3-4周）
### 阶段四：项目实战与面试准备（4-6周）

---

## 🎯 阶段一：基础巩固与框架精通（4-6周）

### 第1-2周：Java核心基础强化

#### 学习目标
- 深入理解JVM原理
- 掌握多线程并发编程
- 熟悉Java集合框架源码
- 理解设计模式应用

#### 每日学习安排（每天4-6小时）
**上午（2-3小时）：理论学习**
- JVM内存模型、GC算法、调优实战
- 并发编程：synchronized、volatile、CAS、AQS原理
- 集合框架源码分析（HashMap、ConcurrentHashMap、ArrayList等）

**下午（2-3小时）：实践编码**
- 手写线程池
- 实现生产者消费者模式
- 手写LRU缓存
- 实现单例模式的多种方式

#### 实践项目
1. **手写简易JVM监控工具**：监控堆内存、GC情况
2. **并发安全工具类**：实现线程安全的计数器、限流器

#### 学习资源
- 《深入理解Java虚拟机》（周志明）
- 马士兵JVM课程
- 并发编程网相关文章

#### 验收标准
- [ ] 能解释JVM内存结构、GC原理
- [ ] 能手写线程池和常见并发工具
- [ ] 能分析HashMap、ConcurrentHashMap源码
- [ ] 能说出至少10种设计模式及应用场景

---

### 第5-6周：数据库与缓存

#### 学习目标

## 🎯 阶段四：项目实战与面试准备（4-6周）

### 第17-18周：综合项目实战

#### 项目选择建议
选择一个**完整的、有亮点的**项目，建议：

**选项1：高并发秒杀系统**
- 技术栈：Spring Boot + Spring Cloud + Redis + MySQL + Kafka
- 亮点：高并发处理、分布式事务、缓存优化、消息队列

**选项2：分布式电商系统**
- 技术栈：微服务架构 + 分布式事务 + 消息队列 + 搜索引擎
- 亮点：微服务拆分、服务治理、分布式系统设计

**选项3：企业级权限管理系统**
- 技术栈：Spring Security + JWT + RBAC + 多租户
- 亮点：安全设计、权限模型、多租户架构

#### 项目要求
1. **代码质量**：规范、注释、单元测试
2. **技术深度**：使用高级特性，解决实际问题
3. **文档完善**：README、架构设计文档、部署文档
4. **可部署**：能实际运行，最好部署到云服务器

#### 项目文档准备
- 项目介绍文档
- 技术架构设计文档
- 核心功能说明
- 技术难点解决方案

- 部署文档

### 第21-22周：面试实战与复盘

#### 面试策略

**1. 投递策略**
- 先投小公司练手，积累面试经验
- 再投目标公司
- 每天投递10-20份简历

**2. 面试技巧**
- STAR法则回答项目问题（Situation、Task、Action、Result）
- 遇到不会的问题，诚实说明，但展示学习能力
- 主动引导话题到熟悉的领域
- 准备反问问题（技术栈、团队、发展）

**3. 面试复盘**
- 记录每次面试的问题
- 总结不会的问题，及时学习
- 优化回答方式

#### 薪资谈判

**准备**
- 了解市场行情（30K对应的要求）
- 准备自己的价值点（技术能力、项目经验）
- 设定薪资底线和期望值

**谈判技巧**
- 不要先报底价
- 强调技术能力和项目经验
- 可以要求股票、期权等
- 考虑综合福利（五险一金、年假等）
